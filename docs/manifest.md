📜 Манифест проекта: BareLog (или MISRA-Log) 

Видение: Создать легковесную, строго типизированную и соответствующую MISRA C библиотеку логирования для STM32, которая предоставляет полный контроль над аппаратным обеспечением без накладных расходов абстракций высокого уровня (HAL/LL). 

Миссия: Предоставить разработчикам встраиваемых систем инструмент для отладки и мониторинга, который: 

     Минимален: Имеет нулевые или минимальные накладные расходы на уровне выполнения и компиляции.
     Безопасен: Полностью соответствует руководящим принципам MISRA C:2012/2023 для предотвращения неопределенного поведения и повышения надежности кода  .
     Ясен: Имеет чистый, документированный API и простой, логичный код для легкого понимания и аудита.
     Гибок: Позволяет легко настраивать уровни логирования, бэкенды вывода (UART, SWV, память) и формат сообщений во время компиляции.
     

Ключевые принципы: 

    Bare-Metal-only: Никаких зависимостей от HAL, LL, RTOS или стандартной библиотеки C (кроме минимально необходимого subset). 
    Прямой доступ к регистрам: Взаимодействие с периферией STM32 через CMSIS-совместимые определения регистров  . 
    Статическая аллокация: Запрет на динамическую память (malloc/free) в соответствии с MISRA C  . 
    Компиляция-время фильтрации: Уровни логирования и форматирование обрабатываются на этапе компиляции для устранения мертвого кода  . 
    Тестируемость: Код спроектирован с учетом модульного тестирования. 

📌 Фаза 1: Минимально жизнеспособный продукт (v0.1) 

Цель: Создать базовый каркас, который может выводить отформатированное сообщение через UART в основном цикле (polling mode). 

     Deliverables:
        API макросов: LOG_DEBUG, LOG_INFO, LOG_WARN, LOG_ERROR. Реализованы через макросы для включения/выключения на этапе компиляции. 
        Кольцевой буфер: Простая, не ISR-безопасная реализация фиксированного размера в оперативной памяти. 
        Простейший UART-бэкенд: Инициализация UART через регистры (CMSIS), передача одного символа/строки в режиме опроса. 
        Базовый формат: "[УРОВЕНЬ] Файл:Строка: Сообщение". 
     

📌 Фаза 2: Ядро функциональности (v0.5) 

Цель: Сделать библиотеку пригодной для реального времени, обеспечив безопасный вывод из прерываний (ISR). 

     Deliverables:
        ISR-безопасный кольцевой буфер: Реализация с использованием volatile указателей и атомарных операций (или критических секций)  . 
        UART-бэкенд с DMA: Настройка DMA для автоматической передачи данных из буфера, минимально загружая CPU. 
        Система уровней: Гибкая настройка максимального уровня логирования во время компиляции. 
        Поддержка фильтрации: Возможность отключать логирование для отдельных файлов (модулей)  . 
     

📌 Фаза 3: Зрелость и соответствие стандартам (v0.9) 

Цель: Добиться промышленного качества, надежности и соответствия MISRA C. 

     Deliverables:
        Статический анализ и MISRA C: Полный прогон кода через анализаторы (например, PC-lint Plus, Helix QAC). Исправление всех нарушений MISRA C:2012  . 
        Дополнительные бэкенды:
             SWO/ITM: Вывод отладочных сообщений через аппаратный отладчик (отлично для bare-metal).
             Память: Возможность сохранять логи в выделенную область Flash-памяти для последующего чтения.
              
        Улучшенный формат: Добавление меток времени (на основе таймера) и поддержки форматирования printf-style (ограниченный subset). 
        Модульные тесты: Базовый набор тестов для кольцевого буфера и форматирования. 
     

📌 Фаза 4: Стабильный релиз (v1.0) 

Цель: Подготовить проект к широкому использованию. 

     Deliverables:
        Полная документация: Подробное описание API, руководство по настройке для STM32CubeMX, примеры использования. 
        Система сборки (CMake): Для легкой интеграции в проекты на основе GCC. 
        Примеры: Минимальный проект для популярных плат (Nucleo, Discovery). 
        CI/CD Pipeline: Автоматическая сборка, запуск статического анализатора и тестов при каждом коммите  . 
        Лицензия и распространение: Выбор перmissive лицензии (например, MIT, Apache-2.0).
